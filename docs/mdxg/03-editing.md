# Editing（編集）

このセクションは、プレビュー / ソースの表示モードと、ドキュメント間のリンクを規定する。

## 11. Mode Toggle（モード切り替え）

[MUST] 準拠ツールは少なくとも 2 つのモードをサポートし、それらを切り替える手段を提供する：

### 11.1 Preview Mode（プレビューモード）

- [MUST] コンテンツ領域は、アクティブページのレンダリング済み HTML のみを表示する（ドキュメント全体ではない）
- [MUST] ページタイトルは目立つように表示される
- [MUST] 標準的な markdown 要素は描画される: 段落、見出し、リスト、コードブロック、テーブル、引用、画像、リンク、水平線
- [MUST] ページ内見出し（H3–H6）は安定したリンク可能な識別子を持つ

### 11.2 Source View（ソースビュー）

- [MUST] コンテンツ領域は **ドキュメント全体** の生の markdown テキストを表示する（アクティブページだけではない）
- [SHOULD] markdown 構文要素には構文ハイライトが提供される: 見出し、太字、斜体、インラインコード、フェンス付きコードブロック、リンク、引用、リストマーカー
- [MAY] テキストは読み取り専用であってもよい

### 11.3 Source Edit（ソース編集）

[MUST] 準拠 MDXG Editor（Conformance を参照）は、ソースビューに加えて編集可能なソースモードをサポートする：

- [MUST] テキストは編集可能である
- [MUST] 編集内容は背後のドキュメントへ同期される

### 11.4 Toggle Behavior（切り替えの挙動）

- [MUST NOT] モードの切り替えは新しいタブ、ウィンドウ、パネルを開いてはならない。遷移はその場で起こる
- [MUST] アクティブモードは明確に示される
- [MAY] ページナビゲーションとページアウトラインはソースモードで利用可能なままでもよい
- [MAY] ページナビゲーションとページアウトラインはソースモードで隠されてもよい。これは実装上の判断

## 12. Document Links（ドキュメントリンク）

[SHOULD] ユーザーが他の markdown ファイルへのリンク（例: `[see spec](SPEC.md)`）をアクティベートしたとき、準拠ツールは、プレーンテキストビューや外部アプリケーションへフォールバックするのではなく、同じ MDXG 準拠ビューワー / エディタでそのファイルを開く。

### 12.1 要件

- [SHOULD] リンクがフラグメントを含む場合（例: `SPEC.md#virtual-pages`）、ツールは対象ドキュメント内の参照された見出しまたはページへナビゲートする
- [SHOULD] 対象ファイルが存在しない、または開けない場合、ツールはエラーをユーザーに示す
- [MUST NOT] クラッシュしてはならず、空の状態へ無言で遷移してもならない
- [MUST] リンクは現在のドキュメントの位置を基準に解決される
- スコープに入るのはローカル markdown ファイルへのリンクのみ
- [SHOULD] 外部 URL（例: `https://...`）はホスト環境のデフォルトの挙動で扱われる
