# Extensions（拡張）

このセクションの機能は SHOULD レベルである。読書体験を強化するが、標準 CommonMark 文法の一部ではない。実装は、実現可能な範囲でこれらをサポートすることが推奨される。

## 14. Math Rendering（数式描画）

[SHOULD] インライン数式と表示数式は、数式組版エンジンを用いて描画される。

### 14.1 要件

- [SHOULD] `$...$` で囲まれたインライン数式は、周囲のテキストとインラインに描画される
- [SHOULD] `$$...$$` で囲まれた表示数式は、中央寄せのブロックとして描画される
- [MUST] 数式描画がサポートされていない場合、生の文法はそのまま保持される
- [MUST NOT] ストリップしたり、隠したり、文字化けさせたりしてはならない
- [SHOULD] 数式の描画スタイルはホストのフォントサイズと色スキームに適応する

### 14.2 実装例

- 高速なクライアントサイド描画のための KaTeX
- より広範な LaTeX カバレッジのための MathJax
- SVG または MathML へのサーバサイド描画

## 15. Diagram Rendering（ダイアグラム描画）

[SHOULD] ダイアグラム言語識別子を持つフェンス付きコードブロックは、プレビューモードで視覚的なダイアグラムとして描画される。

### 15.1 要件

- [SHOULD] 最低限、` ```mermaid ` ブロックはサポートされる
- [MAY] その他のダイアグラム言語（例: `plantuml`、`d2`、`graphviz`）もサポートされてよい
- [MUST] ある言語についてダイアグラム描画がサポートされていない場合、ブロックは標準の構文ハイライト付きコードブロック（セクション 2）にフォールバックする
- [SHOULD] 描画されたダイアグラムはホストの色スキームに適応する

### 15.2 実装例

- クライアントサイド描画のための Mermaid.js
- CLI ツール経由のサーバサイド描画
- ズーム / パン付きのインタラクティブダイアグラム

## 16. Footnotes（脚注）

[SHOULD] 脚注文法は、後方参照付きのリンクされた脚注として描画される。

### 16.1 要件

- [SHOULD] 脚注参照（`[^1]`、`[^note]`）は、脚注定義へジャンプする上付きリンクとして描画される
- [SHOULD] 脚注定義（`[^1]: ...`）は、各参照への後方リンクとともにページ末に描画される
- [MUST] 脚注がサポートされていない場合、生の文法はそのまま保持される
- [MUST NOT] ストリップしたり、隠したりしてはならない

### 16.2 実装例

- ページ末の脚注セクションへリンクする上付き数字
- hover でのツールチップ / ポップオーバープレビュー
- ワイドレイアウトでのサイドノート
