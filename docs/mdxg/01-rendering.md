# Rendering（描画）

このセクションは、markdown コンテンツがプレビューモードでどのように描画されるべきかを規定する。

## 1. Theming（テーマ）

[MUST] 準拠ツールは、その外観をホスト環境に適応させる。

- [SHOULD] 背景色、前景色、アクセント色、ボーダー色は、ホストテーマまたはオペレーティングシステムから導出される
- [MUST NOT] ツールは、使える外観を得るためにユーザーが色を設定しなければならないようにしてはならない
- [MUST] ライトとダーク双方のホストテーマがサポートされる
- MDXG は色スキーム、フォント、視覚スタイルを定義しない

## 2. Code Block Rendering（コードブロック描画）

[MUST] 準拠ツールは、プレビューモードでフェンス付きコードブロックを構文ハイライトとともに描画する。

### 2.1 要件

- [MUST] 言語識別子を持つフェンス付きコードブロック（例: ` ```js `）は、言語に適した構文ハイライトとともに描画される
- [MUST] 言語識別子を持たないコードブロックも、等幅のプリフォーマット済みテキストとして描画される
- [MUST] 各コードブロックには、ユーザーが 1 アクションでブロックの内容をコピーできる「コピー・トゥ・クリップボード」ボタンが提供される
- [MUST] 構文ハイライトの配色スキームは、ホストのライト / ダークテーマに適応する（セクション 1）

### 2.2 実装例

- トークンベースのハイライトには Shiki、Prism、または highlight.js
- hover で出現するか、ブロックの右上に常時表示される「Copy」ボタン
- コピーボタンと並べて表示される言語ラベル

## 3. Task Lists（タスクリスト）

[MUST] 準拠ツールは、プレビューモードでタスクリスト文法をチェックボックスとして描画する。

### 3.1 要件

- [MUST] `- [ ]` は未チェックのチェックボックスとして描画される
- [MUST] `- [x]` はチェック済みのチェックボックスとして描画される
- [MAY] チェックボックスはインタラクティブ（トグルすると元のソースが更新される）であってもよい
- [MAY] チェックボックスは読み取り専用であってもよい。これは実装上の判断
- [MUST] タスクリストはそのリスト構造とインデントを保持する

### 3.2 実装例

- ネイティブ HTML の `<input type="checkbox">` 要素（disabled または enabled）
- クリックハンドラでソースを編集するカスタムチェックボックスコンポーネント

## 4. Images（画像）

[MUST] 準拠ツールは、プレビューモードで画像を妥当なデフォルトとともに描画する。

### 4.1 要件

- [MUST] 画像はドキュメントコンテンツとインラインに描画される
- [MUST] 相対画像パス（例: `![](./images/diagram.png)`）はドキュメントの位置に対する相対として解決される
- [MUST] 画像はコンテンツ幅に収まるよう制約される
- [MUST NOT] コンテンツ領域を水平方向にあふれてはならない
- [MUST] alt テキストは保持され、支援技術からアクセス可能である
- [SHOULD] 画像のロードに失敗した場合、フォールバックとして alt テキストが表示される

### 4.2 実装例

- `<img>` 要素への `max-width: 100%`
- ファーストビュー以下の画像への遅延読み込み
- 大きな画像へのクリックでズーム

## 5. Tables（表）

[MUST] 準拠ツールは、プレビューモードで markdown のテーブルを描画する。

### 5.1 要件

- [MUST] テーブルは可視のセル罫線、もしくは同等の視覚的分離をともなって描画される
- [MUST] ヘッダ行は本文行と視覚的に区別される（例: 太字、背景色）
- [MUST] markdown で指定された列の整列（`---`、`:---`、`:---:`、`---:`）は尊重される
- [MUST] コンテンツ領域より広いテーブルは水平方向にスクロール可能である
- [MUST NOT] ページレイアウトを破壊してはならない

### 5.2 実装例

- テーブルラッパへの `overflow-x: auto`
- 長いテーブルでのスティッキーヘッダ行
- 小さな画面で列をスタックするレスポンシブテーブルレイアウト
