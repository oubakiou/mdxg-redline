# Document Structure（ドキュメント構造）

このセクションは、ドキュメントの仮想的な構造化、ページ間のナビゲーション、検索を規定する。

## 6. Virtual Pages（仮想ページ）

### 6.1 Page Boundaries（ページ境界）

[MUST] 準拠ツールは、ドキュメントを H1 (`#`) と H2 (`##`) の見出し境界で仮想ページに分割する。

[MUST] ATX 形式 (`# Heading`) と setext 形式（`===` または `---` で下線を引いたもの）の両方の見出しを認識する。

[MUST NOT] フェンス付きコードブロック（` ``` ` または `~~~`）の中の見出しはページ境界として扱われない。

[MUST] ツールは見出しをスキャンする際にコードフェンスの open / close 状態を追跡する。

ページ境界をトリガする各見出しが新しいページを開始する。見出しのテキストがページタイトルとなる。次のページ境界までの、見出しに続くコンテンツがそのページに属する。

### 6.2 Pre-heading Content（見出し前のコンテンツ）

[SHOULD] 最初の H1 または H2 見出しよりも前に現れるコンテンツは、"Introduction" というタイトルの暗黙のページとして扱われる。

[SHOULD] 最初の見出しの前のコンテンツが空、または空白のみの場合、Introduction ページは作成されない。

[MUST] Introduction ページが作成される場合、それは他のページとともにページナビゲーションに現れる。

### 6.3 Page Depth（ページの深さ）

各ページは深さを持つ：

- H1 見出しは深さ 1 のページを生成する
- H2 見出しは深さ 2 のページを生成する

[SHOULD] 深さはページを列挙するときに階層を伝えるために使われる（例: インデント、グルーピング、視覚的ウエイト）。

### 6.4 Page Identifiers（ページ識別子）

[MUST] 各ページは、そのタイトルから派生した一意かつ URL セーフな識別子（"スラッグ"）を持つ。

[MUST] 複数のページが同じスラッグを共有する場合、ツールはそれらを曖昧性解消する（例: 数字のサフィックスを付ける）。

## 7. Page Navigation（ページナビゲーション）

[MUST] 準拠ツールは、ユーザーが次のことを行える手段を提供する：

- **すべてのページを見る**: ドキュメント順に、深さ階層が可視な形で
- **任意のページに移動する**: 選択することで
- [MUST] **現在のページを識別する**: アクティブなページが視覚的または文脈的に区別される
- [MUST] **逐次に移動する**: 次ページへ進む、前ページへ戻る、ができる（詳細要件はセクション 9 を参照）

### 7.1 実装例

以下はページナビゲーションがどう実装され得るかの非規範的な例：

- ページのクリック可能なリストを持つサイドバー
- ツールバー内のドロップダウン / セレクトメニュー
- ページ検索を持つコマンドパレット
- モバイルでのスワイプジェスチャー
- キーボードショートカット（例: 矢印キー）
- 折りたたみ可能なツリービュー

## 8. Page Outline（ページアウトライン）

[MUST] 準拠ツールは、ユーザーが現在のページ内の見出しを見て、そこへナビゲートできる手段を提供する。

### 8.1 要件

- [MUST] アクティブページ内の H3 から H6 までの見出しのみが含まれる（H1 と H2 はページ境界であり、ページ内見出しではない）
- [MUST] 各見出しはナビゲート可能である。選択するとその見出しまでスクロール / ジャンプする
- [SHOULD] 見出しの深さは視覚的に伝えられる（例: インデント、フォントサイズ）
- [SHOULD] 現在可視の見出しは示される（スクロールスパイ、もしくは同等）
- [MAY] アクティブページに H3–H6 見出しが無い場合、アウトラインは非表示または空でもよい

### 8.2 実装例

- 右サイドバーの「On this page」リスト
- フローティングアウトラインパネル
- 目次ドロップダウン
- パンくず風の見出しインジケータ

## 9. Sequential Navigation（逐次ナビゲーション）

このセクションは、セクション 7 の逐次移動要件を詳述する。

[MUST] 準拠ツールは、現在のページから前のページ / 次のページに移動する手段を提供する。

### 9.1 要件

- [SHOULD] 前ページと次ページのタイトルはユーザーに可視である
- [MUST] 適用できないコントロール（例: 最初のページでの「previous」）は隠されるか無効化される
- [MUST] 逐次ナビゲーションはインターフェース内の少なくとも 1 箇所からアクセス可能である（例: ツールバー、フッター、キーボードショートカット）
- [MAY] 逐次ナビゲーションは複数箇所からアクセス可能であってもよい

### 9.2 実装例

- ツールバーの前/次ボタン
- ページタイトルと方向インジケータをともなうフッターリンク
- キーボードショートカット
- スワイプジェスチャー

## 10. Search（検索）

[MUST] 準拠ツールは、ドキュメント内の全ページを横断してテキストを検索する手段を提供する。

### 10.1 要件

- [MUST] ユーザーは検索を起動できる（例: Cmd/Ctrl+F または検索アイコン）
- [MUST] 検索はユーザーに可視なレンダリング後のテキストコンテンツに対して動作する。生の markdown ソースに対してではない（例: "bold" の検索が、アスタリスクを介して `**bold**` にマッチしてはならない）
- [MUST] 現在のマッチはハイライトされ、ビューにスクロールインされる
- [MUST] ユーザーはマッチを順に辿れる（次 / 前）
- [MUST] マッチを辿るとき、現在のマッチ位置はページ境界を跨いでも保持される
- [MUST] 他ページのマッチへ移動する操作は、そのページの最初のマッチではなく、その特定のマッチへ着地する
- [SHOULD] マッチ件数は表示される（例: "3 of 12"）
- [SHOULD] ページ境界を跨ぐ検索は、マッチを含むページへ自動的にナビゲートする

### 10.2 実装例

- コンテンツ領域の上部に現れる検索バー
- レンダリング済みコンテンツ内のハイライト `<mark>` 要素
- 次 / 前のマッチへのキーボードショートカット（Enter / Shift+Enter）
