# データ取得 API 仕様書

## 概要

このドキュメントは新規データ取得 API の設計仕様をまとめる。クライアントは認証済みユーザーのプロフィール情報を取得できる。

## エンドポイント

- `GET /api/users/me` — 現在のユーザー情報を取得
- `POST /api/users` — 新規ユーザーを作成

## レスポンス形式

すべてのエンドポイントは JSON を返す。エラー時は HTTP status 4xx/5xx と `{ "error": "..." }` の形式を使う。

## 認証

JWT トークンを `Authorization: Bearer <token>` ヘッダで送る。
